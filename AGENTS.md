# AGENTS.md

Notes for agent sessions working in this repo. Read once, act fast.

## Stack

Client-only **vanilla JS** PWA. No framework, no build step, no bundler, no TypeScript.
Plain HTML/CSS/JS files served statically. Single `index.html` page with sections toggled via class toggles.

- Persistence: **localStorage** (key `copamatch26_my_album_code`) — stores the album as a compact binary code string.
  Grid view preferences (hide-completed / auto-expand-missing) persist separately under `copamatch26_grid_prefs` (JSON).
- PWA: `manifest.json` + `sw.js` (service worker, stale-while-revalidate caching).
- Styling: `style.css`, dark theme, glassmorphism, mobile-first. CSS custom properties (`--accent-primary`, `--accent-secondary`, `--brand-gradient`, `--font-display`).
- Icons: generated by `generate_icons.js` (Node + `canvas` package — the only npm dep).

## Commands

```bash
# Run parser tests (Node.js, uses assert — no test framework)
```

## Sticker model (immutable)

994 stickers total, never change the numbering without bumping `StickerParser.VERSION`:

| Range | Section | Count |
|---|---|---|
| 1–20 | FWC (FIFA World Cup) | 20 |
| 21–34 | CC (Coca-Cola) | 14 |
| 35–994 | 48 teams × 20 stickers | 960 |

Teams are defined in `js/parser.js` → `TEAMS` array (48 entries, 12 groups A–L, 4 teams each).
ISO codes for flag images live in `TEAM_ISO` (used for `flagcdn.com/<iso>.svg`).

`StickerParser.getStickerInfo(id)` maps ID → `{code, flag, name, group, relativeNumber}`.
`StickerParser.getIdFromInfo(code, relativeNumber)` is the inverse.
FWC relative numbers: `00` (id 1), then `1`–`19` (ids 2–20). CC: `1`–`14` (ids 21–34). Teams: `1`–`20`.

## Album code format (binary hybrid)

`StickerParser.generateAlbumCode(state)` → base64url string. Three binary modes, auto-selected by size:

1. **Mode 1 — Bitmask** (default): 125-byte bitmap of owned stickers + repeats list.
2. **Mode 2 — Owned ranges**: list of `[start,end]` pairs (4 bytes each) for owned stickers.
3. **Mode 3 — Missing ranges**: inverse — list of missing ranges, everything else is owned.

The encoder picks the smallest representation. Mode is byte 0 of the binary buffer.
Repeats are always stored as `id:qty` (3 bytes each, qty capped at 255).

**Legacy text format** (`SA26|1|owned-list|repeated-list`) is still parsed for backward compat — `parseAlbumCode` detects it by the presence of `|`. Do not remove legacy parsing.

**Text list format** (free-form lists like `MEX 🇲🇽: 1, 2, 5`) is also supported — detected by keywords `faltante`/`repetida`/`tenho` or `CODE: numbers` line pattern. This is fragile regex parsing; test changes with `js/test_parser.js`.

## DoS prevention

Both parsers have explicit DoS guards — do not remove them:
- Text parser: ranges exceeding `TOTAL_STICKERS` are clamped.
- Binary parser: `numRanges` capped at 1000 (`if (numRanges > 1000) numRanges = 1000;`).

Tests 10–11 in `js/test_parser.js` verify these guards. Any change to parsing must keep all 14 tests green.

## Architecture

```
index.html        Single-page app. 5 sections (tab-section): my-album, matching,
                  independent-compare, stats, import. Share modal. All inline in
                  one HTML file. Nav is a dropdown menu (`menu-trigger-*` buttons).
js/parser.js      StickerParser module: parsing, encoding/decoding, matching.
                  Works in browser (window.StickerParser) and Node (module.exports).
js/app.js         UI controller: state, rendering, event handling, localStorage.
                  Single DOMContentLoaded → init() → loadMyAlbumFromStorage() →
                  renderStickerGrid() → updateMyAlbumUI() → bindEvents() → checkQueryParams().
style.css         Dark theme, glassmorphism, mobile-first. All styling lives here.
sw.js             Service worker. CACHE_NAME (currently 'stickers-swap-v4') —
                  bump on deploys with changed assets or users will get stale content.
manifest.json     PWA manifest. Relative start_url.
generate_icons.js Node script to regenerate icon-192.png / icon-512.png.
js/test_parser.js Node script — the test suite. Run after any parser change.
js/check_user_input.js  One-off verification script for a specific user input.
```

## State shape

```js
state.myAlbum = {
  albumId: 'SA26',
  version: '1',
  owned: new Set([4, 21, 24, ...]),      // sticker IDs the user has (exactly one copy)
  repeated: new Map([[24, 1], [42, 3]]), // sticker ID → extra repeat count
}
```

A sticker that is "owned" is also implicitly in `owned` (repeat count is additional).
`repeated` entries imply `owned` — incrementing a repeat auto-adds to `owned`.

## Matching logic

`StickerParser.matchAlbums(stateA, stateB)`:
- **give** = stickers in A's `repeated` that B does NOT have in `owned`.
- **receive** = stickers in B's `repeated` that A does NOT have in `owned`.

Symmetric. Both lists sorted ascending by sticker ID.

## Independent comparison (two third-party albums)

Section `section-independent-compare` lets a user paste **two other people's** album
lists (not their own) and match them against each other — same `matchAlbums()` logic,
just called with two parsed third-party states instead of `state.myAlbum` + partner.
Handled by `calculateIndependentMatch()` / `renderIndependentMatchResults()` in `app.js`.

## Grid view toggles

`gridPrefs = { hideCompleted, autoExpandMissing }` in `app.js`, persisted as JSON to
`localStorage["copamatch26_grid_prefs"]`. Team sections default to collapsed;
`autoExpandMissing` keeps incomplete teams open, `hideCompleted` adds a
`completion-hidden` class to fully-owned teams. Toggled via `#btn-toggle-hide-completed`
/ `#btn-toggle-auto-expand`, re-rendered through `renderStickerGrid()`.

## URL query params

`?partner=<base64url-code>` — pre-fills the partner code textarea and auto-runs matching on load (`checkQueryParams()` in `app.js`).
Share links are generated in `app.js` → `btnShareImport` / `btnShareCompare` handlers.

## Hard rules

- **No build step, no bundler.** Edit `.html`/`.js`/`.css` directly. Do not introduce webpack/vite/esbuild.
- **No framework.** Do not add React/Vue/Svelte. DOM manipulation is vanilla in `app.js`.
- **Relative paths only** (`./`) — GitHub Pages subpath requires it.
- **Parser is dual-environment.** `js/parser.js` must work in both browser and Node. Keep the `if (typeof module !== 'undefined' && module.exports)` guard at the bottom.
- **Run `node js/test_parser.js` after any parser change.** All 14 tests must pass.
- **`generateFullText(stateParam)` must use `stateParam.repeated`, not `state.myAlbum.repeated`.** The function converts a parsed album into text for the import/compare textarea — using the receiver's `state.myAlbum.repeated` silently drops repeat quantities (bug fixed 2026-06-29).
- **Bump `sw.js` `CACHE_NAME`** when changing cached assets, or users get stale content.
- **Never change the 994-sticker numbering** without bumping `StickerParser.VERSION` and adding backward-compat parsing.
- `.agent/` and `artifacts/` are agent scratch dirs (not source) — keep them out of commits.
